ThoughtWallet v0.0.3 2015.08.01

ThoughtWallet is an HTML5 bitcoin wallet - it runs in the web browser on the local device. The TOR (Firefox) browser is recommended. (ThoughtWallet project on github).

Tips/donations can be made by sending coins to 'Alice', 'Bob', or 'Mary' (also multisig, any combinations of M and N).

A multi-signature transaction output is recognized as spendable if M addresses are in the wallet. A new multisig output can be created by selecting 'Outputs New' then 'Multisig'.

Memo transaction outputs can be created by selecting 'Outputs New', then 'Memo'.

Brainwallet passphrases can be used as substitutes for private keys. ThoughtWallet does not write any data to a server or the local device, so private keys will not automatically be saved. It is important that passphrases or private keys be memorized or written down and stored securely.

Using ThoughtWallet on a quarantined (secure offline or "air gapped") computer:

SETUP. Save this app (save web page, complete) to a thumb drive or SD card.

SYNC. Enter wallet addresses (not keys) and select Data|SyncFromNet, then use Data|Export to save the wallet data to a text file on the thumb drive/SD card.

SIGN. Move the thumb drive/SD card to a quarantined device and open the saved version of this app in the browser there. Use Data|SyncFromFile to import the saved wallet data file. Enter the private keys for the addresses, then build and sign new transactions. Save the raw (hexcode) versions of the transactions back to the thumb drive/SD card.

PUSH. Move the thumb drive/SD card back to the online device and broadcast the raw signed transactions to the bitcoin network using any of the transaction push services available, such as blockchain.info/pushtx.


To load or import data from local files when using Chrome, the browser needs to be run with its --allow-file-access-from-files command line option (the switch instructs Chrome to use Firefox's behaviour regarding local files).
